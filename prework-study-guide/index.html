HTML

  • HTML elements are usually case sensitive and composed of opening tags and closed tags represented by angled brackets.
  • HTML attributes define additional information about an element and are always defined in the beginning element, never the closing element.
  • Elememts that are nested inside other elements are called child elements. The element that hold child elements is known as the parent element.
  • The head element contains information about the webpage.
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage to our user. It is not typically seen but contains important information about accessibility, search engines and performance.
  • The body element represents the visible content shown to the user.

CSS

  • There are three ways to style a webpage using CSS: Incline CSS, Internal CSS stylesheet, External CSS stylesheet.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Important to know that every HTML element is essentially a box consisting of a margin,outside the element,element borders,padding and content.
  • When we want to apply a rule to all elements visable on a page, we use a special wild card symbol.
  • By combining two or more selectors with a comma, declarations can affext them all; this relates to DRY Code.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class sector. Then the class attribute can be assigned to the element in HTML, enabling us to freely choose which elements will be assigned the CSS rule.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add: adds a file as it looks currently to your next commit (stage).
  • git commit -m "descriptive message": commits your staged content as a new commit snapshot.
  • git clone (url): retrieves an entire repository from a hosted location via url.
  • git push (alias) (branch): transmits local branch commits to the remote repository branch.
  • gut pull: fetch and merge any commits from the tracking remote branch.

JavaScript

  • JavaScript is a high level dynamic programming language that allows users to interact with websites.
  • JavaScript is relatively compact yet flexible.
  • A variable is a named container that allows us to store data in our code.
  • To create a variable, you give it a type, a name, and a value.
  • Some examples of different data types that variables may hold include: String, Number, Boolean, Array and Object etc.
  • A comment is a snippet of text that can be added along with code; the broswer ignores comments.
  • A conditional os a code structure used to test if an expression returns tur ot not.
  • A function is way of packaging functionality you wish to use.
  • Functions are a good alternative to repetatively writing the same code.
  • Control flow is the order in which a computer executes code in a script.
  • In JavaScript, code runs from the first statement at the top of the page to the last statement at the bottom of the page.
